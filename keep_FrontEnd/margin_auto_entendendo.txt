1. O que significa "Auto"?
Quando você define margin-left: auto ou margin-right: auto, você está dizendo ao navegador: "Pegue todo o espaço disponível restante e divida-o entre as margens esquerda e direita."
2. A Mágica de margin: 0 auto; 
Top/Bottom (0): Define que não haverá espaço de margem acima ou abaixo.
Left/Right (auto): O navegador pega o espaço total da largura do pai, subtrai a largura do elemento (o seu width) e divide o resultado igualmente entre a esquerda e a direita. 
Exemplo:
Pai: 1000px
Filho (width): 600px
Espaço Restante: 400px (1000 - 600)
margin: 0 auto; -> 400px / 2 = 200px para a esquerda e 200px para a direita. (Centralizado)
3. Por que margin-left: auto empurra pra direita?
Se você definir apenas margin-left: auto e deixar margin-right: 0:
O navegador pega todo o espaço restante e joga apenas para o lado esquerdo.
Resultado: O elemento é empurrado para a direita. 
4. Por que margin-right: auto empurra pra esquerda?
O oposto acontece:
O navegador pega todo o espaço e joga apenas para o lado direito.
Resultado: O elemento é empurrado para a esquerda. 
5. Por que o width e o display são essenciais?
width definido: Se o elemento for um div (que é display: block por padrão), ele tende a ocupar 100% da largura. Se ele já usa 100%, não sobra espaço para a margem automática (auto vira 0). O elemento precisa ser menor que o pai.
display: block ou table: Elementos inline (como span ou a) não respeitam margens laterais automáticas da mesma forma que blocos.
Resumo Visual
css
.elemento {
  width: 50%;
  /* 1. margin-left: auto, margin-right: auto -> Centraliza */
  margin: 0 auto;
}

.direita {
  width: 50%;
  margin-left: auto; /* Empurra da esquerda pra direita */
  margin-right: 0;
}

.esquerda {
  width: 50%;
  margin-right: auto; /* Empurra da direita pra esquerda */
  margin-left: 0;
}
O margin: 0 auto é a forma clássica de centralizar porque equilibra as forças dos dois lados.
