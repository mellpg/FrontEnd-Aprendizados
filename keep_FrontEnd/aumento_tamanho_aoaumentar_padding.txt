POR QUE O PADDING AUMENTA O TAMANHO DO ELEMENTO?

1. O PROBLEMA: O MODELO PADRÃO (content-box)
O aumento inesperado ocorre devido ao "Box Model" padrão do CSS.
Nesse modelo, o padding e a border são somados por FORA da largura definida.

O Cálculo Matemático:
Largura Total = Width + Padding (Esq + Dir) + Border (Esq + Dir)

Exemplo Prático:
Se você define: width: 100px + padding: 20px (cada lado)
O resultado visual será: 140px de largura total.

2. A SOLUÇÃO: O MODELO border-box
Com esta propriedade, o padding e a borda são calculados por DENTRO
da largura total. O elemento mantém o tamanho fixo que você definiu.

O Cálculo com Border-Box:
Largura Total = Width (Fixo)
(O espaço interno para o conteúdo é que diminui).

Exemplo de Código:
.elemento {
box-sizing: border-box;
width: 100px;
padding: 20px;
/* A largura total CONTINUA 100px /
/ O espaço do conteúdo cai para 60px */
}

3. DIFERENÇA CRUCIAL: PADDING VS MARGIN

PADDING (Preenchimento):

Fica dentro do elemento.

Expande a cor de fundo e a área clicável.

MARGIN (Margem):

Fica fora do elemento.

Cria espaço entre um elemento e outro.

Não altera o tamanho da caixa em si.

DICA DE OURO PARA SEUS PROJETOS:
Sempre comece seu CSS com o "Reset Global" abaixo.
Isso torna o layout responsivo muito mais fácil de prever:

{
box-sizing: border-box;
margin: 0;
padding: 0;
}
